<!-- Source: graphthulhu/types/logseq.go + RESEARCH.md Pattern 3 and 4 -->

Polymorphic ref fields (parent, left, page) appear as either:
  {"id": 42}        (full object in read responses)
  42                (bare int in write responses)

PageEntity fields (from logseq.Editor.getAllPages / getPage):
  id, uuid, name, original-name, journal?, journal-day, properties, namespace/name

BlockEntity fields (from getPageBlocksTree):
  id, uuid, content, format, marker, priority
  page: {"id": N} | N
  parent: {"id": N} | N
  left: {"id": N} | N
  children: [BlockEntity, ...] | [["uuid", "val"], ...]   (compact from getBlock)
  properties: dict
  refs: list
  path-refs: list
  pre-block?: bool

Self-referential children: use Field(default_factory=list) + model_rebuild() after class definition.
Compact children from getBlock: [["uuid", "value"]] — silently drop (leave children=[]).
</interfaces>
